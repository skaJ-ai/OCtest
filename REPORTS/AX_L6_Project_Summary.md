# AX 기반 HR 프로세스 마스터 라이브러리 구축 (Phase 1)

## 1. 엔진 기술
본 프로젝트의 데이터 분석 및 생성 로직은 **사내 LLM GAUSS**를 기반으로 수행되었습니다.

## 2. 추진 배경
기존의 행위(Action) 중심 프로세스는 시스템 로그와의 정합성이 낮아 자동화·분석에 한계가 있었습니다.  
본 프로젝트는 업무를 **산출물(Outcome) 중심**으로 재정의하여, AX 자동화의 핵심인 **디지털 트윈(Digital Twin)** 기반을 구축했습니다.

## 3. AS-IS 대비 TO-BE 전환 원칙
- **AS-IS(기존 L6)**: "검토/운영/진행" 중심의 행위 기술이 많아, 실제 시스템 로그와 연결이 약하고 단계 경계가 모호했습니다.
- **TO-BE(신규 L6)**: 모든 L6를 **독립된 결과물(문서/티켓/로그/확정표)** 단위로 정의했습니다.
- **표준 형식**: `[산출물] + [확정/완료/발송/로그 생성]`
- **1:N 분해 원칙**: 기존 1개 행위 L6가 여러 개의 산출물 L6로 분해되더라도, 동일 AS-IS 맥락을 유지해 추적 가능하도록 설계했습니다.

예시)
- AS-IS: 면접운영 준비
- TO-BE:
  1) 면접일정표 확정본 발행
  2) 면접평가표 취합본 등록
  3) 면접전형 결과통보 발송

## 4. 성과 요약
- 전사 **10개 도메인** 대상 전수 정비
- 총 **408개 정밀 L6 데이터셋** 확보
- 원본 대비 해상도 **3배 이상 향상**

## 5. 전략적 가치
- 프로세스 마이닝 시 발생하는 스파게티 맵을 구조적으로 차단
- 디지털 흔적 기반으로 객관적 성과 측정 및 운영 거버넌스 체계 확립
- 이후 L7(BPMN) 설계 및 자동화 시나리오 확장에 즉시 활용 가능
