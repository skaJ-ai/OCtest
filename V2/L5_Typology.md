# L5 업무 유형 사전 (L5 Typology)
**버전**: v1.0 | **작성일**: 2026-03-06 | **기준 CSV**: AX_HR_V2_Master.csv (647행)

---

## 개요

L5는 "시작~종료가 명확한 단위 프로세스"로 정의된다.
각 L5는 **하나의 아키타입(유형)**으로 수렴되며, 이 유형이 표준 L6 흐름의 기준이 된다.

**핵심 원칙**
- L6 동사: 확정/승인/발행(HR판단) · 실시/작성(HR실행) · 발송/배포/취합/추출/등록/접수(DW) · 제공(SSC서비스) · 실시(SSC물리)
- `완료` 동사: **전면 폐기** (V2.1 기준)
- `개최` 동사: **사용 금지** → `실시`로 통합

## 유형별 표준 L6 흐름

### T01 — 제도 수립형
> **트리거**: "가이드 수립", "기준 확정", "계획 수립", "제도 신설", "운영방안 결정" 등

```
표준 흐름:
① [기초 데이터·현황] 추출/취합     → DW (자동 집계·조회)
② [기준·가이드·계획]안 확정        → HR (판단·책임)
③ (승인 필요 시) 품의 승인         → HR
④ [가이드·계획] 배포/발송          → DW (전체 공람 or 대상자 발송)
⑤ (시스템 셋팅 필요 시) 시스템 등록 → DW
```
- ①~② 사이 HR 검토 단계 추가 가능
- 외부 배포 시: `배포`, 개인 발송 시: `발송`
- 판단 없이 시스템 셋팅만: DW 1행으로 처리

### T02 — 선발/심사형
> **트리거**: "서류심사", "면접", "선발 전형", "심사", "후보군 확정" 등

```
표준 흐름:
① [대상자·후보 명단] 추출          → DW
② [평가·심사] 결과 취합             → DW or HR (평가지 디지털화 시 DW)
③ [합격자·선발 대상자] 명단 확정    → HR (최종 판단)
④ 결과 안내문 발송                  → DW
```
- 장소 세팅 필요 시: SSC "장소 세팅 실시" 추가
- 심사 자체가 HR 수행 시: ②→ HR "심사 결과 취합"

### T03 — 면담/상담형
> **트리거**: "면담", "상담", "코칭", "피드백 면담", "1:1" 등

```
표준 흐름:
① [면담 대상자] 명단 확정           → HR
② 면담 일정 안내문 발송             → DW
③ [면담명] 실시                     → HR (대면 대화)
④ 면담 일지/기록 작성               → HR (수기 관리)
⑤ 결과 취합                         → HR
```
- 면담 준비 문서(질문지 등)가 별도 L5인 경우 T01로 처리

### T04 — 행사/이벤트형
> **트리거**: "행사", "미팅", "워크샵", "협의회", "타운홀", "Culture Week" 등

```
표준 흐름:
① [행사·회의] 운영 계획 확정        → HR
② 참가 안내문 발송                  → DW
③ [행사명] 실시                     → HR
④ 결과 보고서 작성/취합             → HR
```
- 회의체(정기협의회·임원회의)는 ① 안건 확정 → ③ 실시 → ④ 회의록 확정

### T05 — 교육 과정 운영형
> **트리거**: "과정 운영", "교육 실시", "양성 운영", "진단/검정 운영" 등

```
표준 흐름:
① 교육 대상자 명단 확정             → HR
② 교육 참가 안내문 발송             → DW
③ [교육명] 실시                     → HR
④ 교육 설문 결과 취합               → DW
⑤ 교육 결과 보고서 작성             → HR
```
- 물리 준비 포함 시: "강의장 세팅 실시(SSC)" 별도 행 추가 (교육운영 L5 '운영준비' 분리)
- 파견·양성 과정 관리도 이 유형 적용

### T06 — 처우/발령 결정형
> **트리거**: "발령", "처우", "이동", "전배", "휴직", "퇴직", "계약", "온보딩" 등

```
표준 흐름:
① [대상자·현황] 추출                → DW
② [처우·이동 조건] 협의·확정        → HR
③ (결재 필요 시) 승인              → HR
④ [안내·결과] 발송                  → DW
⑤ [발령·계약] 시스템 등록          → DW
```
- 퇴직/휴복직: 신청 접수(DW) → 적정성 검토 확정(HR) → 행정안내 발송(DW) → 발령 등록(DW)
- 처우 협의 포함 시: ② "처우 협의 결과 확정(HR)" 유지

### T07 — 모집/공고형
> **트리거**: "Job Posting", "채용 소싱", "대상자 모집", "공고" 등

```
표준 흐름:
① [공고·포스트] 확정               → HR
② [공고문] 배포/발송               → DW
③ 신청·지원 접수                   → DW
④ [적격 후보·지원자] 명단 확정      → HR (1차 필터링 기준 적용)
```
- 이후 선발 단계는 T02로 이어짐 (J/P는 T07+T02 복합 흐름을 T07 하나로 통합 처리)

### T08 — 데이터 처리/등록형
> **트리거**: "시스템 등록", "정보 관리", "매핑", "DB 관리" 등 DW 중심

```
표준 흐름:
① 신청·요청 접수                   → DW (자동 수집 or 채널)
② (예외 처리 필요 시) 검토 확정     → HR
③ [데이터·정보] 시스템 등록        → DW
```
- HR 판단이 없는 단순 등록: DW 1~2행
- 안내 발송이 포함된 경우: 발송(DW) 추가

### T09 — 모니터링/보고형
> **트리거**: "현황 분석", "지표 관리", "결과 보고", "서베이 분석", "데이터 분석" 등

```
표준 흐름:
① [현황·원시 데이터] 추출          → DW
② [분석 결과·인사이트] 취합         → HR (해석·판단)
③ [보고서] 확정                    → HR
```
- 단순 현황 추출 후 배포: ①→발송(DW) 2행으로 축약 가능

### T10 — 캠페인/문화형
> **트리거**: "캠페인", "문화", "Change Agent", "상호존중", "ER 프로그램" 등

```
표준 흐름:
① [캠페인·프로그램] 운영 계획 확정  → HR
② 홍보 자료 배포                   → DW
③ 참여 현황 취합                   → DW
④ 결과 보고서 작성                 → HR
```
- 대상자가 특정되면 ① 이전에 "대상자 명단 확정(HR)" 추가

### T11 — 협상/교섭형
> **트리거**: "단체교섭", "교섭창구", "단체협약", "분쟁조정", "쟁의행위", "협의" 등

```
표준 흐름:
① [협상·교섭 자료] 취합             → DW or HR
② [협의·교섭] 실시                  → HR (대면 협의)
③ [합의·결과] 확정                  → HR
④ (공식 문서 필요 시) 합의서/협약서 발행 → HR
⑤ 시스템 등록                       → DW
```
- 교섭 단계가 반복될 경우: ②~③ 반복 또는 단계별 분리

### T12 — 서비스 운영형
> **트리거**: 총무 인프라 전반, "운영", "지원", "관리" (SSC 기반 지속 서비스)

```
표준 흐름:
① [서비스 운영 기준] 확정           → HR
② 신청·이용 접수                   → DW
③ [서비스명] 서비스 제공            → SSC
```
- 기준이 이미 확정된 루틴 서비스: ② 접수(DW) → ③ 제공(SSC) 2행만
- 현황 보고 필요 시: "운영 현황 확정(HR)" 추가

### T13 — 물류/시설 준비형
> **트리거**: "세팅", "준비", "발송(물리)", "포장", "설치" 등 SSC 1회성 물리 작업

```
표준 흐름:
① (준비 항목 목록 있는 경우) 운영 체크리스트 확정  → HR or DW
② [준비·설치·배송 작업] 실시        → SSC
```
- 단독 물리 작업: SSC 1행만 (강의장 세팅, 교보재 준비, 쿠키 발송 등)

## 유의 사항

### T01 vs T09 구분
- T01: **아웃풋이 "기준/가이드/계획"** — 향후 다른 프로세스의 기준이 됨
- T09: **아웃풋이 "현황 보고서"** — 현재 상태 파악·분석으로 끝남
- 경계 사례: "리텐션" (방안 확정 = T01) vs "동향파악" (현황 취합 = T09)

### T02 vs T06 구분
- T02: **다수 후보 중 "선별/탈락"** — 합격/불합격 결정
- T06: **특정 개인의 "조건·이동"** — 개인별 처우, 발령, 퇴직 처리
- 경계 사례: 승격 후속 조치 (발령 등록 포함 → T06)

### T04 vs T05 구분
- T04: **행사·회의** — 커리큘럼 없음, 1회성 집체
