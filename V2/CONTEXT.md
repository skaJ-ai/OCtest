# V2 설계 컨텍스트 (CONTEXT)

> 이 문서는 CSV 작성 중 판단 기준이 흔들릴 때 참조하는 기준서다.
> PLAN·INVENTORY보다 우선 적용된다.

## 1. 계층 정의

| 계층 | 역할 | 핵심 판정 질문 |
|---|---|---|
| **L3** | 도메인 — 조직의 고유 R&R 단위 | 우리 조직의 고유 전문 영역인가? |
| **L4** | 프로세스 그룹 — 기능 중심 묶음 | 직관적인 '수납장 이름'인가? |
| **L5** | 단위 프로세스 — 시작~종료가 명확한 완결 흐름 | 스톱워치를 켜고 끌 수 있는 한 건의 마디인가? |
| **L6** | 성과 활동 — 원자적 실행 단위 (1 Output, 1 Trace) | 종료 시 [목적어+결과상태] 증거가 남는가? |

## 2. TO-BE L6 표기 공식

```
[결과물 명사] + [L6 표준 동사]
```

### 2-1. 결과물 명사 — 실물 중심

| ❌ 금지 (추상어) | ✅ 허용 (실물) |
|---|---|
| 데이터, 내역, 사항, 내용, 현황, 결과 | 명단, 기준안, 기안서, 결과표, 안내문, 신청서, 계획서, 보고서, 가이드, 계약서, 발령서 |

**L6 결과물 명사 ≠ L5 명칭 반복 금지**

L6는 L5를 원자 단위로 분해한 것이므로, L6 결과물 명사가 상위 L5 명칭을 그대로 반복하면 분해가 아닌 동어반복이 된다.

- **고유명사·복합명사 → 일반명사 추상화**: L5에 특정 브랜드·고유명사가 있으면 L6에서는 일반명사로 바꾼다
  - L5: 타운홀미팅 운영 → L6: **행사** 실시 (타운홀미팅 X)
  - L5: GSAT 코딩 테스트 자격 검증 → L6: **검정** 장소 세팅 실시 (GSAT X)
- **일반명사 반복**: L5와 같은 일반명사를 L6에 쓰는 것은, 해당 L5 그룹 내 L6가 여러 개일 때만 허용. L6가 단 1개뿐이라면 분해 실패로 간주하고 재설계한다

### 2-2. L6 표준 동사 — 마일스톤 중심

| 상황 | 동사 | 예시 | 주체 |
|---|---|---|---|
| 내부 의사결정·버전 고정 | `확정` | 채용규모 확정 | HR |
| 결재권자 공식 사인 | `승인` | 휴직 발령 승인 | HR |
| 외부·타조직 공식 문서 발행 | `발행` | 평가 결과 보고서 발행 | HR |
| 가이드·기준 배포 | `배포` | 임직원 가이드 배포 | DW |
| 메일·알림 송신 | `발송` | 합격 안내문 발송 | DW |
| 여러 출처 데이터 모아 합치기 | `취합` | 부서별 수요조사 취합 | DW |
| 시스템·DB 조건 조회·뽑기 | `추출` | 대상자 명단 추출 | DW |
| 데이터 정합성 검증·비교 | `대조` | 서류 원본 대조 | SSC/DW |
| 시스템 신규 입력 | `등록` | 입사발령 등록 | DW |
| 요청·신청 수령 | `접수` | 휴직 신청 접수 | DW |
| 사람이 직접 수행하는 1회성 실행 활동 | `실시` | 면담 실시, 타운홀 실시, 강의장 세팅 실시 | HR / SSC |
| 수기 문서·일지 완성 | `작성` | 면담 일지 작성, 결과 보고서 작성 | HR |
| SSC 계약·기간 단위 서비스 운영 | `제공` | 피트니스 서비스 제공, 출장 서비스 제공 | SSC |

**`확정` vs `승인` 구분 기준**
- `확정`: 담당자·팀 선에서 버전을 고정. 결재선 불필요 (예: 가이드 확정, 일정 확정)
- `승인`: 상위 결재권자의 공식 사인이 필요 (예: 휴직 승인, 채용 품의 승인)

**`취합` vs `추출` 구분 기준**
- `취합`: 사람/부서에서 보내온 응답을 모으는 것 (예: 각 팀 수요조사 취합)
- `추출`: 시스템에서 조건으로 뽑아내는 것 (예: Workday 대상자 명단 추출)

**`실시` 사용 기준**
- **사람이 직접 수행하는 1회성 실행 활동** — HR(면담·행사·교육)과 SSC(현장 세팅·물리 작업) 모두 사용 가능
- `개최`는 `실시`로 통합 (`개최` 단어 사용 금지)
- DW 행에는 사용 금지 (DW는 자동화 처리 → 발송·등록·취합 등 사용)

**`실시` 오용 방지 규칙 — 핵심**
- 앞에 오는 명사가 **우리 L6 표준 동사 목록에 없는 순수 결과물·행위 명사**일 때만 사용
- ✅ 허용: 면담, 타운홀, 행사, 교육, 세팅 — L6 표준 동사 목록에 없는 순수 명사
- ❌ 금지: 앞 명사가 이미 L6 표준 동사 목록에 있는 동사성 명사인 경우
  - "안내 발송 실시" (X) → "안내 발송" (O) — `발송`이 이미 L6 표준 동사
  - "수요조사 취합 실시" (X) → "수요조사 취합" (O) — `취합`이 이미 L6 표준 동사
  - "명단 확정 실시" (X) → "명단 확정" (O) — `확정`이 이미 L6 표준 동사
- **판단 기준**: "앞 명사가 L6 표준 동사 목록에 있는가?" → YES면 `실시` 불필요, NO면 `실시` 사용 가능

**`작성` 사용 기준**
- 시스템 없는 환경에서 수기로 문서·일지를 완성하는 행위
- 시스템이 있으면 `등록` 사용 / 시스템 없으면 `작성` 사용
- 예: 면담 일지(수기) → `작성` / 면접 평가표(Workday) → `등록`

**`제공` 사용 기준**
- SSC가 계약·기간 단위로 서비스를 지속 운영·완료하는 활동
- `[서비스명] 서비스 제공` 형태로 작성 (예: "피트니스 서비스 제공")
- SSC 전용. HR·DW 행 사용 금지
- 운영 결과에 대한 HR 보고는 별도 행으로 분리: `[운영] 결과 보고서 확정 (HR)`

**`완료` 사용 금지**
- `완료`는 공식 L6 표준 동사가 아님. L6 표기에 사용하지 않는다.
- 모든 L6 표준 동사는 완료를 이미 내포하고 있으므로 `완료` 추가 불필요
- SSC 1회성 물리 작업 → `실시` 사용 (예: "강의장 세팅 실시")
- SSC 지속 서비스 → `제공` 사용 (예: "피트니스 서비스 제공")
- DW·HR → `발송`, `등록`, `확정`, `승인`, `실시`, `작성` 등 해당 동사 사용

### 2-3. 금지 패턴 (V1.0 잔재 — 등장 시 무조건 재작성)

- `요건 확인용 기초 자료 취합`
- `타당성 검토 및 최종 결과안 확정`
- `확정 결과 시스템 반영 및 저장`
- `운영 일정 및 가용 자원 리스트 취합`
- `최종 운영 시나리오 및 타임테이블 확정`
- `수신 대상자 그룹 명단 추출`
- `안내문 초안 톤앤매너 검수 및 확정`
- `기초 현황표 시스템 추출`
- `최종 의사결정 및 현장 조치 확정`
- `[AS-IS L6명]` 브래킷 prefix 일체
- DW·HR 행에 `완료` 동사 사용

## 3. 계층별 변경 정책

| 계층 | 변경 가능 여부 | 기준 |
|---|---|---|
| **L3** | **절대 불가** | AS-IS 그대로 |
| **L4** | **원칙 불가, 예외 허용** | 타 L3와 계층 일관성을 맞추기 위한 명백한 사유가 있을 때만 분리 |

| **L6** | **전면 재설계** | V1.0 무시, 실무 결과물 기반으로 새로 정의 |

## 4. 실행 주체 분류 기준

### HR
- 대면 협상, 예외 판단, 최종 기준 확정·승인
- 정성적 판단이 필요한 의사결정
- 면담·협의 활동 (DW로 쪼개지 않음)
- **판단 기준**: "규칙만으로 결정 불가능한가?" → YES이면 HR

### Digital Worker (DW)
- 규칙 기반 필터링·추출·계산
- 시스템 등록 (Workday 등)
- 템플릿 기반 대량 메일·알림 발송
- 챗봇 응대, 자동 접수
- **판단 기준**: "규칙과 데이터만으로 자동화 가능한가?" → YES이면 DW

### SSC
> ⚠️ **보수적 접근 필수** — SSC 이관은 비용이 크고 실행 난이도가 높다.

- **핵심 원칙: 횡단 묶음 이관** — 동일/유사한 성격의 L6가 여러 L5·L4에 반복 등장할 때, 그 유사 L6 전체를 한꺼번에 SSC로 넘기는 방식
- L5 내 일부 L6만 SSC여도 무방 — 단 다른 L5/L4의 유사 L6와 반드시 함께 묶어야 함
- L5 전체가 물리적 성격이면 L5 단위 통이관도 가능
- **판단 기준**: "이 L6와 유사한 성격의 업무가 다른 L5/L4에도 반복 존재하는가?" → YES이면 SSC 후보

**SSC 후보 L6 유형 (횡단 반복)**
- 현장 세팅/준비류: 강의장 세팅, 검정 장소 준비, 면접실 준비
- 물리적 발송/배송류: 교보재 배송, 퀵·택배·DHL 발송, 웰컴키트 배송
- 서류 수령·보관류: 제출 서류 수령, 서명 서류 보관

### Eliminate
- 수기 엑셀 취합 (시스템으로 대체 가능)
- 불필요한 중간 보고 (대시보드로 대체 가능)
- 관행적으로 남아있는 중복 확인 절차
- **판단 기준**: "없애도 결과물 품질에 영향 없는가?" → YES이면 Eliminate

## 5. 분해 패턴 5가지

### Pattern A — 의사결정/확정형 (HR 중심)
> 적용 대상: 명단 확정, 타당성 심사, 대상자 선정, 처우 결정 등

**기본 구조 (2행)**
```
1. [판단 근거 자료] 취합/추출  →  DW
2. [결과물] 확정/승인          →  HR
```

**3행으로 확장하는 기준**: HR 확정 후 시스템에 별도 등록 단계가 명확히 존재할 때
```
1. [판단 근거 자료] 취합/추출  →  DW
2. [결과물] 확정/승인          →  HR
3. [결과물] 등록               →  DW
```

**1행으로 줄이는 기준**: 판단 근거 취합이 HR 행위에 이미 포함되어 분리 불필요할 때
```
1. [결과물] 확정               →  HR
```

**예시**
- `부서별 채용 수요조사 취합` (DW) → `전사 채용규모 확정` (HR)
- `면접위원 후보 명단 추출` (DW) → `면접위원 명단 확정` (HR) → `면접위원 명단 등록` (DW)

---

### Pattern B — 알림/발송형 (DW 중심)
> 적용 대상: 공지, 안내 메일, 리마인더, 결과 통보 등

**1행 기준**: 발송 대상이 명확하거나 전사 공지처럼 고정된 경우
```
1. [안내문/메일] 발송  →  DW
```

**2행 기준**: 발송 대상이 조건 필터링 필요하거나, 문구가 민감해 HR 검수 필요한 경우
```
1. [수신 대상자 명단] 추출       →  DW
2. [안내문/메일] 발송            →  DW
```
또는 민감 안내(인사발령·징계 등):
```
1. [안내문 초안] 확정            →  HR
2. [안내문] 발송                 →  DW
```

**예시**
- `서류 합격 안내문 발송` (DW, 1행) — 전형 결과 자동 발송
- `복직 안내문 발송` (DW, 1행) — 일정 기반 자동 트리거
- `징계결과 통보문 확정` (HR) → `징계결과 통보문 발송` (DW) — 민감 안내

---

### Pattern C — 기획/분석형 (HR+DW 협업)
> 적용 대상: 보고서 작성, 전략 수립, 가이드 수립, 분석 결과 도출 등

**기본 구조 (3행)**
```
1. [기초 데이터/현황] 취합/추출  →  DW
2. [분석 결과] 취합              →  HR  (인사이트 정리)
3. [기안서/가이드/보고서] 확정   →  HR
```

**2행으로 줄이는 기준**: 분석과 문서화가 HR에서 하나로 이어지는 경우
```
1. [기초 데이터] 취합/추출  →  DW
2. [보고서/가이드] 확정     →  HR
```

**주의**: `인사이트 도출` 표현 금지. 결과물 명사가 없는 행위 동사임.
→ 대신 `[결과 보고서] 확정`, `[분석 결과] 취합` 형태로 작성

**예시**
- `평가 실시율 명단 추출` (DW) → `평가 결과 보고서 확정` (HR)
- `인력 분포 현황 추출` (DW) → `인력 현황 분석 결과 취합` (HR) → `인력계획 기준안 확정` (HR)

---

### Pattern D — 물리적/운영형 (SSC 횡단 묶음 이관)
> ⚠️ 보수적 적용. 단독 L6 하나만 SSC로 보내지 않는다.
> 적용 대상: 여러 L5/L4에 걸쳐 반복되는 동일 성격의 물리적 L6 묶음

```
[유사 성격 L6 묶음] 운영 완료  →  SSC
```

- 단일 L5 내 1개 L6만 SSC → 원칙적으로 불가
- 동일/유사 성격 L6가 여러 L5에 반복 → 전체 묶어 이관 → 가능
- L5 전체가 물리적 성격이면 L5 단위 통이관도 가능

**예시 (적용 가능)**
- 교육운영 강의장 세팅 + 자격검증 검정 장소 준비 + 면접 장소 준비 → 동일 성격 → SSC 묶음 이관

**예시 (적용 불가)**
- 면접 장소 준비만 단독 SSC → 불가 (다른 L5와 묶지 않은 경우)

### Pattern E — Sparse 도메인 추론형
> 적용 대상: L6가 없는 6개 도메인 (보상/근태, 노사, 총무, 해외인사, 기획, 임원조직)
> AS-IS L6 컬럼에 `(데이터 없음)` 표기

```
L5명 분석 → 업무 성격 판단 → 위 A~D 패턴 중 적합한 것 적용
```

도메인별 주체 성향:
| 도메인 | 주체 성향 | 이유 |
|---|---|---|
| 보상/근태 | DW 비중 높음 | Workday 기반 자동화 |
| 노사 | HR 비중 높음 | 대면·협의 중심 |
| 총무 | SSC 비중 높음 | 물리적 현장 운영 (단, SSC 횡단 묶음 원칙 동일 적용) |
| 해외인사 | HR 비중 높음 | 글로벌 복잡성 |
| 기획 | HR 중심, DW 지원 | 전략·보고 |
| 임원조직 | HR 거의 전부 | 고도 판단 |

> ⚠️ **ER·노사 영역 특칙**: 가급적 최대한 HR 주체로 반영. 휴먼터치(대면 협의·공감)는 DW 대체 불가 고부가가치 활동이므로 DW 억지 분할 금지.

### Sparse 도메인 실행형 마디 유형 (L5 성격별 분해)

L5 성격을 먼저 판별한 뒤 해당 유형 패턴을 적용한다.

#### 유형 1 — 행사형 (타운홀, 시무식, 시상식, 기념행사 등)
```
1. [행사] 계획서 확정        →  HR
2. [행사] 안내 발송          →  DW
3. [행사명] 실시             →  HR
4. [행사] 결과 보고서 작성   →  HR
```

#### 유형 2 — 면담형 / Take-care형 (마음건강, 고충 면담, 육아지원 상담 등)
```
1. 면담 대상자 명단 확정     →  HR
2. 면담 실시                 →  HR
3. 면담 일지 작성            →  HR  ← 수기 관리 시 `작성`
4. 면담 결과 취합            →  HR
```

#### 유형 3 — 캠페인형 (상호존중문화, 인식 제고 캠페인 등)
```
1. 캠페인 계획서 확정        →  HR
2. 홍보 자료 배포            →  DW
3. 참여 현황 취합            →  DW
4. 캠페인 결과 보고서 작성   →  HR
```

#### 유형 4 — 운영관리형 (STEP, 정기 프로그램, 사내 제도 운영 등)
```
1. 대상자 명단 확정          →  HR
2. 일정 안내 발송            →  DW
3. [프로그램] 운영 실시      →  HR
4. 운영 결과 취합            →  HR
```

#### 유형 5 — 여성ER 2트랙형 (시스템+면담 병행)
**트랙 A — 시스템 기반 (DW 중심)**
```
1. 대상자 명단 추출          →  DW
2. 프로그램 안내 발송        →  DW
3. 참여 결과 취합            →  HR
```
**트랙 B — 면담 기반 (HR 중심)**
```
1. 면담 대상자 명단 확정     →  HR
2. 면담 실시                 →  HR
3. 면담 일지 작성            →  HR
4. 면담 결과 취합            →  HR
```

## 9. 채점 루브릭 (TO-BE L6 품질 검증)

### 9-1. 이중 채점 구조

L6 표기의 주체 판단은 **동사 단독으로 불가능**하다. 결과물의 성격이 1순위, 동사는 보조 힌트다.

| 우선순위 | 채점 항목 | 판단 질문 | 결과 |
|---|---|---|---|
| **1순위** | 결과물 성격 (판단 복잡도) | "이 결과물을 만들기 위해 규칙 외 인간 판단이 필요한가?" | YES → HR / NO → DW 후보 |
| **2순위** | 동사 범주 | "이 동사가 L6 표준 동사 목록 어느 카테고리에 속하는가?" | HR판단형→HR / DW처리형→DW / SSC서비스·실행형→SSC |

**오류 패턴**: 동사 범주만 보고 주체를 판단하면 오류 발생.
- `확인 보고서 확정` (HR판단형 동사) → "확인 보고서"가 자동화 가능한 단순 대조면 DW가 맞음
- `대상자 명단 추출` (DW처리형 동사) → "대상자 선정 기준"이 예외 판단을 요하면 HR이 맞음

### 9-2. 결과물 판단 복잡도 기준

| 복잡도 | 결과물 특징 | 예시 | 주체 |
|---|---|---|---|
| **고 (HR)** | 규칙 외 판단, 예외 처리, 면담·협의 산물 | 채용규모 확정안, 평가 등급 기준안, 면담 일지, 전략 보고서 | HR |
| **중 (HR+DW)** | 민감 내용 포함 or 복합 단계 | 인사발령 안내문(민감), 협력사 계약서(협의 필요) | HR 확정 → DW 발송 |
| **저 (DW)** | 조건 기반 자동화 가능 | 대상자 명단(조건 필터), 안내 메일(템플릿), 시스템 등록 | DW |
| **물리 (SSC)** | 현장 실행·시설 운영 | 강의장 세팅, 웰컴키트 발송, 시설 서비스 운영 | SSC |

### 9-3. 빈출 오류 패턴 (채점 시 확인)

| 오류 유형 | 오류 표현 | 올바른 표현 |
|---|---|---|
| 동사성 명사 + 실시 | "안내 발송 실시" | "안내문 발송" |
| 완료 suffix | "명단 취합 완료" | "명단 취합" |
| L5 명칭 반복 | L5: 타운홀 운영 → L6: "타운홀 운영 확정" | "타운홀 실시" |
| 추상어 결과물 | "기초 자료 취합", "현황 확인" | "수요조사 취합", "접속 현황 추출" |
| 인사이트 표현 | "인사이트 도출 완료" | "결과 보고서 확정" |
| 단계 표현 | "1차 검증", "최종 확정" | 결과물 명사 + L6 표준 동사 |

### 9-4. HR 담당자 공개 기준

내부 CSV·L6 표준 동사 전체 목록·채점 루브릭은 **비공개**. 담당자에게는 아래만 공개한다.

**공개 항목**
1. **L6 공식**: `[결과물 명사] + [L6 표준 동사]`
2. **결과물 명사 금지 목록**: 데이터, 내역, 사항, 내용, 현황, 결과
3. **금지 표현 목록**: `~진행`, `~완료` suffix, L5 명칭 반복, 추상어 결과물

**비공개 이유**: 동사 목록·루브릭을 사전 공개하면 담당자가 결과물 성격 판단 없이 동사만 선택하는 단순 매핑 오류 유발. 담당자가 L6를 직접 작성한 후 루브릭으로 채점하는 방식이 품질이 높다.
